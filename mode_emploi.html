1. Présentation

La Simulation de Loge RER est un outil pédagogique interactif permettant de visualiser le déroulement complet d'une Tenue au Rite Écossais Rectifié, sur les quatre grades symboliques (Apprenti, Compagnon, Maître, Maître Écossais de Saint-André).

Elle représente le plan de loge vu de dessus, avec les pions des Officiers et des Frères, anime leurs déplacements selon le rituel, restitue les dialogues et actions cérémoniales en temps réel, et accompagne le tout de sons rituels et de musiques de fond.

Elle intègre également un jeu de quiz interactif (XQuiz RER) permettant de tester ses connaissances du catéchisme, adapté au grade de l'utilisateur.

À quoi sert-elle ?

  • Former les Officiers à leur rôle et à la chorégraphie du rituel
  • Permettre aux Frères de répéter les cérémonies sans mobiliser toute la Loge
  • Servir de référence visuelle pour la mise en place du temple
  • Préparer les réceptions de candidats étape par étape
  • Tester et approfondir ses connaissances du catéchisme via le quiz interactif

Que faut-il pour l'utiliser ?

  • Un navigateur web moderne (Chrome, Firefox, Safari ou Edge, version récente)
  • Aucune installation nécessaire — tout fonctionne en local, sans connexion internet
  • Résolution recommandée : 1 400 pixels de large minimum
  • Les sons et musiques nécessitent que le navigateur autorise la lecture audio

2. Démarrage — Écran de connexion

À l'ouverture du fichier Loge_RER.html dans votre navigateur, un écran de connexion s'affiche sur fond sombre, aux couleurs du Rite.

Étapes de connexion

  1. Sélectionner votre grade dans le menu déroulant (Apprenti, Compagnon, Maître ou Maître Écossais)
  2. Saisir le mot de passe correspondant à votre grade
  3. Cliquer sur « Entrer en Loge » ou appuyer sur Entrée

Le mot de passe est insensible à la casse. La validation fonctionne aussi bien avec le bouton qu'avec la touche Entrée.

Un lien Guide de l'utilisateur est disponible en bas de l'écran de connexion. Il ouvre le guide utilisateur dans un nouvel onglet, sans nécessiter de mot de passe.

Un lien Guide de l'utilisateur est disponible en bas de l'écran de connexion. Il ouvre le guide utilisateur dans un nouvel onglet, sans nécessiter de mot de passe.

Accès par grade

Le grade sélectionné détermine les cérémonies accessibles dans le panneau de régie et le niveau des questions dans le quiz. Un Compagnon a accès à toutes les cérémonies d'Apprenti plus celles de Compagnon, un Maître a accès aux trois grades de Saint-Jean, et un Maître Écossais accède à l'ensemble des cérémonies du 4e grade.

Accès direct par URL

Pour pré-sélectionner un grade, ajoutez le paramètre ?grade=apprenti (ou compagnon, maitre, me) à l'URL. Le mot de passe reste obligatoire.

Mots de passe

3. L'interface

Une fois connecté, l'interface se compose de trois zones principales :

Le panneau de régie (à gauche)

Bande verticale contenant tous les boutons de contrôle du rituel, organisés en sections accordéon. C'est le « poste de pilotage » de la simulation.

Le plan de loge (au centre)

Représentation SVG du temple vu de dessus, orienté avec l'Orient en haut et l'Occident en bas.

Le flux des travaux (à droite)

Panneau déroulant qui transcrit en temps réel l'intégralité du rituel : dialogues, actions cérémonielles, informations système.

Le panneau catéchisme (optionnel)

Activable depuis la section Système, ce panneau intermédiaire affiche le vocabulaire du grade courant.

Panneaux escamotables

Les panneaux de régie et du flux peuvent être repliés grâce à leurs boutons de bascule. La largeur du panneau flux est ajustable par glisser-déposer.

Les pions des Officiers peuvent afficher soit un symbole géométrique soit une image de portrait. Le bouton « Bijoux : ON/OFF » permet de basculer.

Tous les pions peuvent être déplacés manuellement par glisser-déposer, y compris pendant une animation. Le glisser-déposer fonctionne sur écran tactile.

6. Le flux des travaux

Le panneau de droite transcrit en temps réel l'intégralité du rituel. Types de contenu :

  • Dialogues des Officiers — texte doré, précédé du nom du locuteur
  • Actions rituelles — texte gris en italique
  • Informations système — texte bleu discret

Le bouton « Effacer le flux » vide l'affichage. Le panneau défile automatiquement. La largeur est ajustable.

7. Le catéchisme

Le panneau catéchisme, activable via « Catéchisme : ON/OFF », affiche les questions et réponses du catéchisme du grade courant. Le contenu est organisé en sections correspondant aux sections du catéchisme rituel.

8. Simuler une Tenue ordinaire

Ordre recommandé :

  1. Entrée des Frères
  2. Ouverture des travaux (grade concerné)
  3. Catéchisme et/ou Instruction morale (facultatif)
  4. Clôture des travaux

Attendre systématiquement « En attente » avant de lancer la séquence suivante.

Raccourci : Loge prête — place instantanément tous les pions en position de Tenue ouverte.

9. Simuler une Tenue de réception

Ordre recommandé :

  1. Entrée des Frères
  2. Ouverture des travaux
  3. Proclamation → Introduction → Voyages → Serment → Réception → Lumière → Investiture
  4. Clôture des travaux

Les étapes doivent être lancées dans l'ordre (6 pour l'Apprenti, 7 pour les autres grades).

Formation de la Loge

Lors de l'Introduction, tous les Frères se rassemblent en formation autour du tapis. Un voile sombre recouvre momentanément le plan.

Les voyages

Le candidat effectue trois voyages autour du temple, accompagné du MC et de l'Introducteur. Chaque voyage correspond à une épreuve élémentaire (Feu, Eau, Terre). Le plan est assombri.

Pause et reprise

La cérémonie peut être suspendue à tout moment via Pause ou Espace.

Interruption et reset

« Ranger la loge » (ou Échap) interrompt et remet tous les pions.

10. Sons et musique

Sons rituels

Coups de maillet synthétisés par Web Audio API. Applaudissements synthétisés.

Musiques de fond

Fichiers MP3 dans imusic/, démarrage/arrêt automatique. Bouton « Musique : ON/OFF ».

Principe de fonctionnement

15. Le moteur d'animation (moteur.js)

15.1 Animation programmée — animerVers()

Signature :

animerVers(elementId, cibleX, cibleY, duree = 1000)
  • Récupère l'élément SVG par ID, lit position via transform.baseVal
  • requestAnimationFrame pour animation fluide 60 FPS
  • Easing ease-in-out (Bézier quadratique)
  • Respecte Rituel._paused
  • Retourne Promise résolue à destination

15.2 Drag & Drop

  • startDrag(evt) — détecte clic sur .draggable
  • drag(evt) — met à jour translate
  • endDrag() — libère l'élément
  • Conversion via svg.getScreenCTM(). Tactile supporté.

20. Gestion des grades et conformité

20.1 Système de conformité

miseEnConformite(grade) adapte le plan. Objet _CONFORMITE définit par grade : tapis, tableau, étoile flamboyante (C), miroir (C), éléments M, éléments ME, vertus progressives (Justice/Clémence toujours ; +Tempérance C ; +Prudence M).

20.2 Modes visuels

Épique (couleur, matières) / Classique (N&B simplifié). Basculement via toggleLogeMode().

20.3 Transitions entre grades

Sortie des Frères du grade inférieur, changement de conformité, retour éventuel.

21. Système de pause

Rituel._paused respecté à 3 niveaux :

  1. Animations (moteur.js) — temps figé
  2. Délais (scenarios.js) — ticks de 50 ms conditionnels
  3. Attente (_attendrePause) — bloque l'exécution

23. Authentification

23.1 Mécanisme

Hash SHA-256 en JS pur. Constante HASHES. Mot de passe en minuscules avant hachage.

23.2 Modifier un mot de passe

  1. Ouvrir Loge_RER.html dans un navigateur
  2. Console F12, taper : sha256("nouveau_mot_de_passe")
  3. Copier le hash
  4. Ouvrir le HTML dans un éditeur, chercher HASHES, remplacer le hash du grade
  5. Sauvegarder

25. Maintenance et évolution

25.1 Sauvegarde

Fichiers dans Dropbox (historique des versions).

25.2 Ajouter une musique

  1. Placer le MP3 dans imusic/
  2. Dans scenarios.js : jouerMusique('fichier.mp3')
  3. arreterMusique(true) pour fondu, arreterMusique() pour coupure sèche

25.3 Ajouter un pion

  1. Dans Loge_RER.html : ajouter <g class="draggable" id="...">
  2. Dans scenarios.js : ajouter dans PION_IDS, POSITIONS, REMISE, EXIST
  3. setExist(id, 1) pour rendre visible

25.4 Modifier un scénario

Convention de code :

async maCeremonie() {
    const self = this;
    self._setEtape("Ma cérémonie");
    await self.dire("VM", "Texte du dialogue");
    await self.processer("pion-mc", 900, WP[0], WP[1], WP[2]);
    await self.action("Description de l'action rituelle");
    self._setEtape("En attente");
}

25.5 Ajouter des questions au quiz

  1. Ouvrir XQuiz RER/questions.js
  2. Ajouter { question, answers, correct } dans la catégorie
  3. Vérifier correct (index base 0)

25.6 Débogage

  • Mode Debug (touche D)
  • Bouton « Positions »
  • Quiz : bouton ⊹ pour voir les réponses
  • Console F12

25.7 Dépendances

Aucune. Polices Google Fonts (Cinzel, Crimson Text, Cinzel Decorative, IM Fell English).
